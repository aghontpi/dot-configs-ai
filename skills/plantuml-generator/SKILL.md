# PlantUML Generator

Stay focused on the diagram task only: choose `generate`, `extract`, or `update`, run the helper script, and return the rendered PNG or extracted PlantUML source. The script handles Java validation, metadata embedding, and the cached PlantUML jar automatically.

## Workflow

1. Decide whether the task is `generate`, `extract`, or `update`.
2. Use `skills/plantuml-generator/scripts/generate.py`.
3. For `generate` and `update`, let the script manage `skills/plantuml-generator/resources/plantuml.jar`. It downloads the jar only when the cache is missing and refresh-checks for newer releases on a longer cadence instead of every run.
4. Re-embed the PlantUML source into the PNG so the diagram stays editable later.

## Exact Commands to Execute

To build or interact with diagrams, execute the appropriate helper script command below. The `generate.py` script ensures that Java version requirements are met and correctly embeds PlantUML source code directly into the PNG metadata, so you should prefer it over manually invoking PlantUML.

**1. Generate a new PNG from a `.puml` file:**
Use this command when you have created or edited a `.puml` file and need to render it into an image.
```bash
python3 skills/plantuml-generator/scripts/generate.py generate <path_to_puml> [output_path_png]
```

**2. Extract embedded PlantUML source from a PNG:**
Use this command to retrieve the hidden source code embedded in a PNG previously generated by this tool.
```bash
python3 skills/plantuml-generator/scripts/generate.py extract <path_to_png> [-o output.puml]
```

**3. Update a PNG using the source already embedded in its metadata:**
Use this command to simply re-render a PNG file without having a separate `.puml` file.
```bash
python3 skills/plantuml-generator/scripts/generate.py update <path_to_png>
```

**4. Update a PNG with a new inline source string:**
Use this command for quick diagram edits by supplying the PlantUML structure directly over the CLI.
```bash
python3 skills/plantuml-generator/scripts/generate.py update <path_to_png> -s "@startuml
Alice -> Bob: hello
@enduml"
```

## Runtime Notes

- `extract` works without Java or the PlantUML jar because it only reads PNG metadata.
- `generate` and `update` require Java 8+.
- The script validates that Java 8+ is available before `generate` or `update`.
- If `java` is not on PATH, the script also checks `PLANTUML_JAVA_CMD`, `JAVA_HOME/bin/java`, and on macOS `/usr/libexec/java_home`.

### Optional overrides

- `PLANTUML_JAR_PATH`
- `PLANTUML_JAR_URL`
- `PLANTUML_JAVA_CMD`
- `PLANTUML_JAR_REFRESH_DAYS`
- `--refresh-days 7` checks weekly, `--refresh-days 30` checks monthly, and `--refresh-days 0` disables periodic refresh checks.

## Output Expectations

- Generated PNGs include a `Source` metadata field with the PlantUML text.
- Extraction can print the source to stdout or write it to a `.puml` file.
- Updates overwrite the PNG in place and refresh the embedded source.

## Troubleshooting

- `PlantUML jar not found`: rerun `generate` or `update` with network access, or provide `--jar-path`.
- `Java 8+ is required`: install a compatible Java runtime, or point the script at one with `--java-cmd`, `PLANTUML_JAVA_CMD`, or `JAVA_HOME`.
- `No PlantUML source found in metadata`: the PNG was not produced by this workflow, or its metadata was stripped.
- `Dot executable does not exist`: Graphviz is missing. Sequence diagrams often still work, but many other diagram types need Graphviz on macOS/Linux.
